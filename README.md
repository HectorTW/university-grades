# Система оценок для школы

Веб-приложение для управления оценками студентов с системой регистрации, авторизации и ролевого доступа.

## 📋 Возможности

### Для студентов:
- ✅ Регистрация и авторизация
- ✅ Создание и редактирование личного профиля
- ✅ Загрузка фотографии профиля
- ✅ Указание формы обучения (очная, заочная, дистанционная)
- ✅ Выбор желаемого направления и специализации
- ✅ Поле "О себе" для описания хобби, достижений и мотивации
- ✅ Просмотр всех своих оценок (⭐ 1–5) по предметам с комментариями учителей
- ✅ Просмотр полученных жетонов и их накопленной статистики (6 типов: Упорство, Трудолюбие, Понимание, Подготовка, Креатив, Скорость)
- ✅ Получение приглашений на собеседования от работодателей
- ✅ Принятие или отклонение приглашений (с появлением контактных данных)

### Для учителей:
- ✅ Регистрация с одобрением администратора
- ✅ Выставление оценок по группам студентов и предметам
- ✅ Оценка в формате ⭐ 1–5
- ✅ Выдача жетонов (0–3 из 6) к каждой выставленной оценке
- ✅ Добавление комментариев к оценкам

### Для работодателей:
- ✅ Регистрация с одобрением администратора
- ✅ Создание профиля компании с полной информацией
- ✅ Просмотр студентов с фильтрацией и сортировкой
- ✅ Просмотр характеристик (0–100), рассчитанных по оценкам и весам предметов (вместо показа “сырых” оценок)
- ✅ Просмотр накопленных жетонов студентов
- ✅ Отправка приглашений на собеседования (без начального доступа к контактам)
- ✅ Просмотр фотографий и ограниченной информации студентов

### Для администраторов:
- ✅ Управление всеми пользователями
- ✅ Одобрение регистраций учителей и работодателей
- ✅ Удаление пользователей
- ✅ Контроль доступа к системе
- ✅ Управление направлениями обучения
- ✅ Управление специализациями и группами
- ✅ Управление характеристиками для работодателя и коэффициентами влияния предметов на характеристики
- ✅ Мониторинг приглашений на собеседования

## 🛠️ Технологии

- **Backend**: Python Flask 2.3.3
- **База данных**: PostgreSQL (с миграциями через Flask-Migrate)
- **Frontend**: HTML5, CSS3, JavaScript, Bootstrap 5
- **Аутентификация**: Flask-Login
- **Безопасность**: Flask-WTF (CSRF), Flask-Limiter (rate limiting), Flask-Talisman (security headers)
- **Валидация**: WTForms, Marshmallow
- **Загрузка файлов**: Pillow (валидация изображений)

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Настройка переменных окружения

Скопируйте `.env.example` в `.env` и настройте значения:

```env
SECRET_KEY=your-secret-key-here
FLASK_DEBUG=True
APP_ENV=development
DATABASE_URL=postgresql://user:password@localhost:5432/school_grades
```

Перед первым запуском:
1. Установите PostgreSQL и создайте базу: `createdb school_grades`
2. Укажите в `DATABASE_URL` пользователя и пароль.
3. Примените миграции: `python -m flask db upgrade`

**Генерация SECRET_KEY:**
```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

### 3. Запуск приложения

```bash
python app.py
```

Приложение будет доступно по адресу: http://localhost:5000

### 4. Первый вход

Администратор **не создаётся автоматически** при запуске (это небезопасно для production).

Для первого запуска в **development** можно включить bootstrap-режим через `.env`:

```env
BOOTSTRAP_DATA=True
BOOTSTRAP_ADMIN_USERNAME=admin
BOOTSTRAP_ADMIN_EMAIL=admin@school.com
BOOTSTRAP_ADMIN_PASSWORD=change-me-strong-password
```

В production оставляйте `BOOTSTRAP_DATA=False` и создавайте админа отдельно (например, через админку/миграции/скрипт).

## 🚢 Деплой (production)

### 1) Переменные окружения

- Используйте `.env.production.example` как шаблон.
- Обязательно установите `APP_ENV=production`, `FLASK_DEBUG=False`, `SECRET_KEY` (случайный 32+ байта hex).
- Включите HTTPS и `SESSION_COOKIE_SECURE=True`.

### 2) Запуск WSGI-сервером

Файл входа: `wsgi.py`.

- **Linux (gunicorn)**:

```bash
gunicorn -w 2 -b 0.0.0.0:8000 wsgi:app
```

- **Windows (waitress)**:

```bash
waitress-serve --host=0.0.0.0 --port=8000 wsgi:app
```

Рекомендуется ставить приложение за reverse proxy (Nginx/Caddy) и включать `FORCE_HTTPS=True`.

## 🗄️ База данных

Приложение использует **PostgreSQL** с следующими таблицами:

- **User** - пользователи системы (студенты, учителя, админы, работодатели)
- **StudentProfile** - профили студентов (включая фото, форму обучения, направления)
- **EmployerProfile** - профили работодателей с информацией о компании
- **StudyDirection** - направления обучения
- **Specialization** - специализации
- **Group** - группы студентов
- **Subject** - предметы
- **Grade** - оценки студентов
- **InterviewInvitation** - приглашения на собеседования

### Миграции базы данных

Проект использует **Flask-Migrate** для управления изменениями в базе данных. Это позволяет безопасно изменять структуру БД без потери данных.

**Основные команды:**
```bash
# Создать миграцию при изменении моделей
python -m flask db migrate -m "Описание изменений"

# Применить миграцию
python -m flask db upgrade

# Просмотреть историю
python -m flask db history
```

📖 **Подробнее**: См. [COMMANDS.md](COMMANDS.md) - раздел "Работа с миграциями"

## 📖 Использование

### Регистрация студентов
1. Перейдите на страницу регистрации
2. Выберите роль "Студент"
3. Заполните форму регистрации (требуется пароль: мин. 8 символов, заглавные буквы, цифры)
4. Студенты регистрируются автоматически

### Регистрация учителей и работодателей
1. Перейдите на страницу регистрации
2. Выберите соответствующую роль
3. Заполните форму регистрации
4. Дождитесь одобрения администратора

### Выставление оценок
1. Войдите как учитель
2. Выберите группу студентов
3. Укажите предмет
4. Выставьте оценки по трем критериям (1-5): оценка, трудолюбие, знания
5. Добавьте комментарий (необязательно)
6. Сохраните оценки

### Администрирование
1. Войдите как администратор
2. Просмотрите список пользователей
3. Одобрите регистрации учителей и работодателей
4. Удалите нежелательных пользователей
5. Управляйте направлениями обучения, специализациями, группами и предметами
6. Мониторьте приглашения на собеседования

## 🔒 Безопасность

Приложение включает следующие меры безопасности:

- ✅ **CSRF защита** - все формы защищены от CSRF атак
- ✅ **Валидация входных данных** - WTForms формы для всех входных данных
- ✅ **Rate limiting** - защита от брутфорса (5 попыток входа в минуту)
- ✅ **Хеширование паролей** - Werkzeug для безопасного хранения паролей
- ✅ **Security headers** - Flask-Talisman для защиты от XSS, clickjacking и т.д.
- ✅ **Безопасная загрузка файлов** - валидация изображений через PIL
- ✅ **Управление сессиями** - Secure, HttpOnly, SameSite cookies
- ✅ **Логирование** - запись критичных действий и ошибок
- ✅ **Парольная политика** - требования к сложности пароля

📖 **Подробнее**: См. [SECURITY.md](SECURITY.md) - полная документация по безопасности

## 🐛 Решение проблем

### Приложение не запускается
- Проверьте, установлены ли все зависимости: `pip install -r requirements.txt`
- Убедитесь, что порт 5000 свободен
- Проверьте переменные окружения в `.env`

### Ошибки миграций
- Убедитесь, что база данных существует
- Попробуйте: `python -m flask db stamp head`
- Проверьте логи в `logs/app.log`

### Проблемы с безопасностью
- Проверьте, что CSRF токены присутствуют во всех формах
- Убедитесь, что SECRET_KEY установлен в `.env`
- Проверьте логи для диагностики
